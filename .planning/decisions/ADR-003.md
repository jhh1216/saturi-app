# ADR-003: Levenshtein Distance 유사도 계산 선택

- **날짜:** 2026-06-02
- **상태:** 승인됨

---

## 배경

음성 인식으로 얻은 텍스트와 정답 방언 문장을 비교하여 0~100점 점수를 계산해야 한다. 비교 알고리즘 후보는 다음과 같다.

1. **완전 일치 (Exact Match)**
2. **Levenshtein Distance (편집 거리)**
3. **자카드 유사도 (Jaccard Similarity)**
4. **코사인 유사도 (TF-IDF 기반)**
5. **외부 NLP API** (의미 유사도)

## 결정

**Levenshtein Distance** 기반 정규화 점수를 선택한다.

```dart
class SimilarityCalculator {
  static int calculate(String target, String input) {
    if (target.isEmpty && input.isEmpty) return 100;
    if (target.isEmpty || input.isEmpty) return 0;

    final int m = target.length;
    final int n = input.length;
    // DP 테이블로 편집 거리 계산
    final List<List<int>> dp = List.generate(
      m + 1, (_) => List.filled(n + 1, 0),
    );
    for (int i = 0; i <= m; i++) dp[i][0] = i;
    for (int j = 0; j <= n; j++) dp[0][j] = j;

    for (int i = 1; i <= m; i++) {
      for (int j = 1; j <= n; j++) {
        if (target[i - 1] == input[j - 1]) {
          dp[i][j] = dp[i - 1][j - 1];
        } else {
          dp[i][j] = 1 + [dp[i-1][j], dp[i][j-1], dp[i-1][j-1]].reduce(min);
        }
      }
    }

    final int distance = dp[m][n];
    final int maxLen = max(m, n);
    return ((1 - distance / maxLen) * 100).round().clamp(0, 100);
  }
}
```

## 이유

### Levenshtein Distance를 선택한 근거

**1. 부분 점수 지원 — 학습 동기 유지에 필수**

사투리 학습 초기에는 완벽한 발음이 어렵다. 완전 일치 방식은 한 글자만 달라도 0점이 된다. Levenshtein은 얼마나 가까운지를 수치화하여 학습자가 점진적 향상을 느낄 수 있게 한다.

예시:
```
정답: "와 진짜 맛있겠다 아이가"
인식: "와 진짜 맛있겠다 아이고"
→ 완전일치: 0점
→ Levenshtein: 약 93점 (1글자 차이)
```

**2. 음성 인식 오류에 관대**

`speech_to_text`의 한국어 인식은 사투리를 표준어에 가깝게 변환하는 경향이 있다. Levenshtein은 받침 하나, 모음 하나 차이에도 높은 점수를 줄 수 있다.

**3. 외부 의존성 없음**

순수 Dart로 구현 가능하다. 외부 패키지나 API 없이 `similarity.dart` 단일 파일로 완결된다.

**4. 계산 복잡도가 앱에 적합**

Levenshtein의 시간 복잡도는 O(m×n)이다. 방언 문장 길이는 평균 10~20자이므로 O(400) 수준 — 모바일 기기에서 수 마이크로초 안에 계산된다.

**5. 이해하기 쉬운 알고리즘**

동적 프로그래밍 기반으로 교과서에 수록된 알고리즘이다. 코드 리뷰와 발표에서 설명하기 용이하다.

### 대안을 선택하지 않은 이유

| 대안 | 기각 이유 |
|---|---|
| 완전 일치 | 1글자 차이에도 0점 → 학습 의욕 저하 |
| 자카드 유사도 | 어절 단위 비교라 짧은 문장에서 정밀도 낮음, 어순 무시 |
| 코사인 유사도 | TF-IDF 벡터화 필요, 짧은 문장에선 의미가 없음 |
| 외부 NLP API | 비용·지연·인터넷 의존성, 오버엔지니어링 |

## 점수 해석 기준

```
90~100점  완벽해요! 진짜 현지인 같아요 🎉
70~ 89점  괜찮아요! 조금만 더 연습하면 돼요 👍
50~ 69점  아쉬워요. 다시 들어보고 따라해봐요 🔄
 0~ 49점  처음부터 다시 해봐요! 💪
```

70점 이상: 성공 효과음 / 미만: 실패 효과음
