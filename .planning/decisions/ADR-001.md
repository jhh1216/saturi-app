# ADR-001: Flutter 선택

- **날짜:** 2026-06-02
- **상태:** 승인됨

---

## 배경

사투리 학습 앱을 개발할 플랫폼을 선정해야 한다. 주요 후보는 다음과 같다.

1. **Flutter** (Dart) — 크로스플랫폼 UI 프레임워크
2. **React Native** (JavaScript) — 크로스플랫폼, JS 기반
3. **Android Native** (Kotlin) — Android 전용
4. **Web App** (HTML/JS) — 브라우저 전용

## 결정

**Flutter**를 선택한다.

## 이유

### Flutter를 선택한 근거

**1. 단일 코드베이스로 Android·iOS·Web 동시 지원**

음성 인식 기능을 가진 앱을 Android와 Web 양쪽에서 시연해야 한다. Flutter는 `flutter build web`, `flutter build apk` 한 명령으로 각 플랫폼 빌드가 가능하다. React Native는 웹 지원이 불완전하고 별도 라이브러리(React Native Web)가 필요하다.

**2. `speech_to_text` 패키지의 Flutter 지원이 가장 성숙**

한국어 음성 인식을 지원하는 패키지 중 `speech_to_text`는 Flutter 전용으로 개발·유지되고 있으며, Android(SpeechRecognizer API), iOS(SFSpeechRecognizer), Web(Web Speech API)를 단일 인터페이스로 추상화한다. 이 패키지의 Flutter 지원 품질이 가장 높다.

**3. Widget 기반 UI의 빠른 프로토타이핑**

`StatefulWidget`, `AnimationController`, `TweenAnimationBuilder` 등 Flutter의 내장 애니메이션 도구로 WaveRing, 점수 카운트업 등 인터랙티브 UI를 짧은 코드로 구현할 수 있다.

**4. Hot Reload로 빠른 UI 이터레이션**

UI 디자인을 여러 차례 수정(단청 → 파스텔 리디자인)해야 했는데, Flutter의 Hot Reload(`r`)로 저장 즉시 결과를 확인할 수 있어 개발 속도가 크게 향상됐다.

**5. Material Design 3 기본 지원**

`useMaterial3: true` 설정 한 줄로 최신 Material You 스타일의 기본 컴포넌트를 사용할 수 있다.

### 대안을 선택하지 않은 이유

| 대안 | 기각 이유 |
|---|---|
| React Native | 웹 지원 불완전, speech_to_text 패키지 품질 낮음, JS 런타임 오버헤드 |
| Android Native | iOS·Web 미지원, 학습 범위 초과 |
| Web App | 모바일 네이티브 기능(마이크 권한, TTS) 연동이 복잡, 앱스토어 배포 불가 |

## 결과

- Flutter 3.x (Dart 3.x) 사용
- `pubspec.yaml`에 `flutter_sdk: ">=3.0.0"` 명시
- 빌드 타겟: Android APK, Web(Chrome)
- 개발 환경: VS Code + Flutter Extension
