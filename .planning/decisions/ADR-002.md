# ADR-002: speech_to_text 패키지 선택

- **날짜:** 2026-06-02
- **상태:** 승인됨

---

## 배경

사투리 발음을 텍스트로 변환하기 위한 음성 인식 방식을 결정해야 한다. 주요 후보는 다음과 같다.

1. **`speech_to_text`** (pub.dev 패키지)
2. **Google Cloud Speech-to-Text API** (외부 REST API)
3. **Web Speech API 직접 구현** (웹 전용)
4. **OpenAI Whisper API** (외부 REST API)

## 결정

**`speech_to_text`** 패키지(`^7.x`)를 선택한다.

## 이유

### `speech_to_text`를 선택한 근거

**1. 디바이스 내장 음성 인식 엔진 활용 — 무료, 오프라인 가능**

`speech_to_text`는 OS 내장 음성 인식을 사용한다.
- Android: `SpeechRecognizer` (Google 음성 서비스)
- iOS: `SFSpeechRecognizer` (Apple 음성 인식)
- Web: `webkitSpeechRecognition` / `SpeechRecognition` API

별도 API 키나 클라우드 비용이 없다. 학교 프로젝트에서 외부 API 비용 부담 없이 운영할 수 있다.

**2. 한국어(`ko_KR`) 공식 지원**

패키지 내에서 `localeId: 'ko_KR'`를 설정하면 한국어 인식이 활성화된다. 사투리 발음도 음소 단위에서 한국어 기반으로 인식된다.

```dart
await _speech.listen(
  localeId: 'ko_KR',
  listenFor: const Duration(seconds: 10),
  pauseFor: const Duration(seconds: 3),
);
```

**3. Flutter 멀티플랫폼 단일 API**

Android·iOS·Web에서 동일한 Dart 인터페이스로 사용할 수 있다. 플랫폼별 분기 코드가 불필요하다.

**4. 실시간 스트리밍 결과 지원**

`onResult` 콜백으로 발화 도중에도 인식 중간 결과를 실시간으로 받을 수 있다. 사용자는 녹음 중에도 인식 텍스트가 화면에 표시되어 진행 상황을 확인할 수 있다.

```dart
onResult: (r) {
  setState(() => _recognized = r.recognizedWords);
  if (r.finalResult) _finish(); // 최종 결과로 채점
},
```

**5. 권한 요청 내장**

`initialize()` 호출 시 마이크 권한을 자동으로 요청한다.

### 대안을 선택하지 않은 이유

| 대안 | 기각 이유 |
|---|---|
| Google Cloud STT API | 유료(월 60분 초과 과금), API 키 관리 필요, 인터넷 필수 |
| Web Speech API 직접 | 웹 전용 → Android·iOS 별도 구현 필요, 코드 중복 |
| OpenAI Whisper API | 유료, 지연시간 높음(실시간 불가), API 키 보안 문제 |

## 한계 및 주의사항

- **웹(Chrome)에서 HTTPS 필요:** `localhost`에서는 동작하지만, 배포 시 반드시 HTTPS 환경이어야 한다.
- **Android에서 인터넷 연결 필요:** Android의 Google 음성 서비스는 클라우드 인식을 사용하므로 네트워크가 필요할 수 있다.
- **사투리 인식 정확도:** 표준어 기반 모델이므로 강한 사투리는 인식률이 낮을 수 있다. Levenshtein distance 채점으로 부분 점수를 주어 보완한다 (→ ADR-003 참고).

## 결과

- `speech_to_text: ^7.0.0` pubspec.yaml에 추가
- Android `AndroidManifest.xml`에 `RECORD_AUDIO` 권한 추가
- iOS `Info.plist`에 `NSMicrophoneUsageDescription` 추가
